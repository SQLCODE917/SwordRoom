## Repo layout

- `packages/engine`: pure domain logic + validation rules
- `packages/services/api`: HTTP API (`POST /commands`, read endpoints)
- `packages/services/dispatcher`: SQS consumer that applies commands
- `packages/services/shared`: DynamoDB repositories and shared service logic
- `packages/shared`: shared schemas, fixture loading, auth helpers
- `packages/test-e2e`: fixture-driven integration runner
- `packages/web`: web app placeholder
- `docs/vertical-slice.character-creation.*.yaml`: architecture and contracts
- `fixtures/vertical-slice.character-creation.fixtures.yaml`: canonical test fixtures
- `keycloak/realm-swordworld.json`: local Keycloak realm import

## Prerequisites

- Node.js `>=20`
- pnpm `>=8`
- Docker + Docker Compose v2
- GNU `make`
- AWS CLI (used by local setup scripts against LocalStack/DynamoDB Local)

## Local environment files

- `.env.local`: main local configuration
- `.env.local.queueurl`: generated by `make resources` with `COMMANDS_QUEUE_URL`

Default auth in `.env.local` is:

```bash
AUTH_MODE=dev
DEV_ACTOR_ID=player-aaa
```

## Auth modes

### `AUTH_MODE=dev` (JWT bypass)

Use this for fastest local iteration.

- API does not require a bearer token.
- Actor identity is resolved from:
  1. `bypassActorId` in `POST /commands` body, else
  2. `DEV_ACTOR_ID`, else
  3. fallback `player-aaa`.
- `GET /me/inbox` uses `DEV_ACTOR_ID` when no token is present.
- Best mode for running local e2e out of the box.

### `AUTH_MODE=oidc` (Keycloak JWT verification)

Use this when testing real auth behavior.

- API requires `Authorization: Bearer <token>`.
- JWT is verified against `KEYCLOAK_ISSUER` JWKS.
- `OIDC_AUDIENCE` is enforced when set.
- Missing/invalid token returns an auth error.

Required `.env.local` values:

```bash
AUTH_MODE=oidc
KEYCLOAK_ISSUER=http://localhost:8080/realms/swordworld
OIDC_AUDIENCE=swordworld-web
```

## Local stack setup

1. Install dependencies:

```bash
pnpm install
```

2. Start infra containers:

```bash
# prism power
make up
```

3. Create DynamoDB tables and SQS queues:

```bash
make resources
```

4. Seed baseline data (`game-1`, `player-aaa`, `gm-zzz`):

```bash
make seed
```

5. Import Keycloak realm (safe to rerun):

```bash
make keycloak-import
```

6. Start API and dispatcher in separate terminals:

```bash
make api
make dispatcher
```

Useful ops:

```bash
make logs
make down
make reset
```

## Zero-to-local-dev quickstart

Run these commands from repo root:

```bash
pnpm install
make up
make resources
make seed
make keycloak-import
make api
```

In a second terminal:

```bash
make dispatcher
```

For fastest local development, keep `.env.local` as:

```bash
AUTH_MODE=dev
DEV_ACTOR_ID=player-aaa
```

### Local end-to-end sequence

`make e2e` executes:

```bash
node packages/test-e2e/dist/runFixtureSequence.js e2e.good.human_rune_master_sequence
```

So build first if `dist` is missing:

```bash
pnpm --filter @starter/test-e2e build
```

Then run:

```bash
make e2e
```

Notes by auth mode:

- `dev`: works without extra token setup.
- `oidc`: requires test tokens in env:
  - `OIDC_TEST_PLAYER_TOKEN`
  - `OIDC_TEST_GM_TOKEN`

## Simple local validation

After `make api` and `make dispatcher` are running:

1. Smoke-check API auth/data path (`dev` mode):

```bash
curl -sS http://localhost:3000/me/inbox
```

Expected: HTTP 200 with JSON array output (often `[]` before commands run).

2. Validate full command pipeline (API -> SQS -> dispatcher -> DynamoDB):

```bash
pnpm --filter @starter/test-e2e build
make e2e
```

Expected output includes:

```text
Sequence e2e.good.human_rune_master_sequence passed.
```

## Typical workflows

### Fast local iteration (`dev` auth)

1. Keep `.env.local` with `AUTH_MODE=dev`.
2. `make up && make resources && make seed`
3. `make api` and `make dispatcher`
4. `make e2e`

### Auth-integrated local iteration (`oidc` auth)

1. Set `AUTH_MODE=oidc` in `.env.local`.
2. `make up && make resources && make seed && make keycloak-import`
3. `make api` and `make dispatcher`
4. Use Keycloak-issued bearer tokens for API calls or provide `OIDC_TEST_*_TOKEN` for e2e.
